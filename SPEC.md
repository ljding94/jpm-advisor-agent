## Assignment Recap
Three agents collaborate to advise a client:
- **Client Agent**: simulated client with profile (age, risk aversion, assets, investments).
- **Advisor Agent**: only agent allowed to talk to both Client and Analyst; defines tasks for Analyst.
- **Analyst Agent**: has access to internet + a knowledge store; fetches info for Advisor.

Routing constraint is hard: **Analyst must not communicate with Client directly.**

## Architecture
LangGraph `StateGraph` with three nodes (one per agent) and conditional edges enforcing routing.

State machine:
`GATHER_PROFILE → ANALYZE → ADVISE → CONFIRM → RESOLVED` (or `TERMINATED` on limit hit)

Design patterns to demonstrate (each must be referenced in README):
1. **Mediator** — Advisor mediates Client ↔ Analyst.
2. **Strategy** — pluggable risk-profile strategies.
3. **Factory** — agent instantiation.
4. **Observer** — structured turn-by-turn logger.
5. **State machine** — explicit status enum.

## Component Specifications

### Schemas (Pydantic v2)
- **`ClientProfile`**: `age: int (18–100)`, `risk_tolerance: Literal["conservative","moderate","aggressive"]`, `assets: dict[str, float]`, `investments: list[Investment]`, `goals: list[str]`, `time_horizon_years: int`, `annual_income: float`.
- **`AgentMessage`**: `sender: AgentRole`, `recipient: AgentRole`, `content: str`, `timestamp: datetime`, `message_type: Enum`, `metadata: dict`. Validator enforces `(sender, recipient)` is in the allowed routing set.
- **`AnalystReport`**: `query: str`, `findings: str`, `sources: list[Source]` (must be non-empty), `confidence: float`.
- **`AdviceOutput`**: `recommendations: list[str]`, `rationale: str`, `sources: list[Source]`, `disclaimers: list[str]` (must include the standard not-financial-advice disclaimer).

### Graph State (`graph/state.py`)
`TypedDict` with: `client_profile`, `conversation_history: list[AgentMessage]`, `current_advisor_query`, `analyst_findings`, `draft_advice`, `status: ConversationStatus`, `turn_count: int`, `errors: list[str]`.

### Agents

**`BaseAgent` (abstract)** — `name`, `system_prompt`, `llm`, optional `tools`; methods `process(state) -> state`, `_call_llm(messages, **kwargs)` with retry + exponential backoff (tenacity).

### `ClientAgent`
- Profile generated by LLM at init from a seed (or loaded from JSON for deterministic tests).
- Answers Advisor questions consistent with profile; confirms/rejects advice.
- On confirmation → sets `status=RESOLVED`.
- No tool access.

**Sample personas** (store as JSON in `data/personas/`):

1. **`margaret_conservative.json`** — Margaret Chen, 62, recently retired teacher. Assets: $850K (60% bonds/CDs, 30% large-cap stocks, 10% cash). No mortgage, no dependents at home. Goal: preserve capital and generate ~$40K/yr income for the next 25 years. Risk tolerance: conservative. Time horizon: 25 years. Concerns: inflation, healthcare costs, outliving savings.
2. **`david_moderate.json`** — David Patel, 42, software engineer. Assets: $500K (70% diversified equities, 20% bonds, 10% cash). Two kids (ages 8 and 11). Mortgage: $280K remaining. Goals: fund both kids' college in 7–10 years, retire at 60. Risk tolerance: moderate. Time horizon: 18 years. Annual income: $185K.
3. **`priya_aggressive.json`** — Priya Sharma, 28, product manager at a tech startup. Assets: $120K (85% equities incl. growth stocks, 10% crypto, 5% cash). No dependents, no mortgage. Goal: aggressive wealth accumulation, optional FIRE by 45. Risk tolerance: aggressive. Time horizon: 17+ years. Annual income: $165K.

The `main.py` runner accepts `--persona {margaret,david,priya}` (default: `david`) and produces a corresponding sample conversation. Integration tests run against all three personas.

**`AdvisorAgent`** (mediator)
- Decides next action: ask client more questions, dispatch task to analyst, synthesize advice, or resolve.
- Output is structured JSON: `{next_action, target, message}`.
- Synthesizes `AnalystReport`s into client-friendly `AdviceOutput`.

**`AnalystAgent`**
- Tools: `web_search`, `knowledge_store_query`.
- Always queries knowledge store first (cheaper); falls back / augments with web search.
- Must return `AnalystReport` with non-empty `sources`.

### Tools
- **`web_search.py`** — DuckDuckGo wrapper using the `ddgs` package. Async, retried (tenacity), returns top 5 results as `[{title, url, snippet}]`. No API key required. Rate-limit polite (sleep 1s between calls). Wrapped behind a `WebSearchProvider` interface so it's swappable.
- **`knowledge_store.py`** — ChromaDB persisted at `data/chroma/`. Ingestion script reads `data/knowledge_base/*.md`, chunks (500 tokens, 50 overlap), embeds via the configured `EmbeddingProvider`, exposes `similarity_search(query, k=4)` returning chunks with `{text, source, score}` metadata.

**`EmbeddingProvider` interface**
- `OpenRouterEmbeddings`: uses `OPENROUTER_API_KEY`, model `openai/text-embedding-3-small` (1536 dims).
- `LocalEmbeddings`: uses `sentence-transformers/all-MiniLM-L6-v2` (384 dims) — no key required, used in tests and as fallback.

Provide 5–10 seed knowledge docs covering: asset allocation basics, risk tolerance frameworks, retirement planning principles, diversification, tax-advantaged accounts. Generate them as markdown if not available.

### Guardrails
- **`pii.py`** — regex for SSN, credit card, account numbers, emails, phone; redact before any LLM call. Log redaction events.
- **`output_filter.py`** — reject outputs that: name specific tickers, contain "guaranteed return"/"risk-free"/"can't lose", or omit disclaimer. Force-append disclaimer if missing.
- **`limits.py`** — `MAX_TURNS=20`, `MAX_TOKENS_PER_CALL=4000`, `MAX_TOTAL_COST_USD=2.00`, per-call timeout 60s. Breach → `status=TERMINATED` with reason.

### Strategy Pattern (`strategies/risk_profile.py`)
Abstract `RiskStrategy` with `recommended_allocation(profile) -> dict[asset_class, pct]`. Concrete: `ConservativeStrategy`, `ModerateStrategy`, `AggressiveStrategy`. Advisor selects strategy based on `client_profile.risk_tolerance`.

### Factory (`factories/agent_factory.py`)
`AgentFactory.create(role: AgentRole, config: dict) -> BaseAgent`. Handles LLM client, tool wiring, system prompt loading.

### Observability (`observability/logger.py`)
Structured JSON logs per turn: `{turn, agent, action, input_tokens, output_tokens, cost_usd, duration_ms}`. Final transcript exporter writes markdown to `examples/sample_conversation.md`.

## Termination
- Advisor emits `status=RESOLVED` when client confirms.
- Hard limits → `status=TERMINATED`.
- Graph END node writes transcript + summary stats.

## Testing (≥70% coverage required)
- **Schema tests**: routing constraint rejects illegal sender/recipient pairs; profile validation.
- **Agent unit tests**: mocked LLM with fixed responses; assert outputs and state mutations.
- **Tool tests**: mocked Tavily client and Chroma collection.
- **Guardrail tests**: positive and negative cases per guardrail (PII detected/missed, banned phrases caught, disclaimer enforcement).
- **Graph tests**: routing function returns correct next node for each status.
- **Integration test**: full conversation with deterministic mocked LLM responses; assert state machine transitions and `RESOLVED` reached.
- **Edge cases**: client supplies contradictory info, analyst tool failure → graceful degradation, infinite loop prevention via `MAX_TURNS`, PII in client message gets redacted before LLM.

Use `pytest`, `pytest-asyncio`, `pytest-cov`. Mock LLM via a `FakeLLM` fixture in `conftest.py`.

## Constraints
- Python 3.11+.
- Pin all dependencies in `requirements.txt`.
- No secrets in code; `.env` only.
- **LLM provider**: OpenRouter by default (`OPENROUTER_API_KEY`, base URL `https://openrouter.ai/api/v1`). Default model `anthropic/claude-sonnet-4` (configurable via `LLM_MODEL` env var). Wrap calls behind an `LLMProvider` interface so users can swap in OpenAI/Anthropic/Azure/local by changing env vars. Use the OpenAI Python SDK pointed at OpenRouter's base URL — it's drop-in compatible.
- **Web search**: DuckDuckGo via the `ddgs` Python package (no API key). Wrap behind a `WebSearchProvider` interface.
- **Embeddings**: OpenRouter `openai/text-embedding-3-small` by default (uses same `OPENROUTER_API_KEY`). Wrap behind an `EmbeddingProvider` interface. Provide `sentence-transformers` (`all-MiniLM-L6-v2`) as offline fallback when `OPENROUTER_API_KEY` is missing.
- If `OPENROUTER_API_KEY` missing: tests use `FakeLLM` and `sentence-transformers`; `main.py` prints a clear setup message and exits.

## Out of Scope
- No web framework (no FastAPI/Flask) — CLI only.
- No Docker.
- No real market data feeds.
- No multi-conversation persistence.
- No UI.

## Deliverable
A self-contained git repo. A reviewer with only `OPENROUTER_API_KEY` set should be able to clone, install, run tests, and run `python -m src.main --persona david` to see a complete advisor conversation. (Web search requires no key.)
